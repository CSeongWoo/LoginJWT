# LoginJWT

## 설명에 앞서
- 이 프로젝트는 싱글톤과 이 개념이 스프링 프레임워크에서 어떻게 활용되는지 이해하기 위한 학습용 프로젝트입니다.
- 실제 로직을 만들어 스프링과 싱글톤 패턴의 관계를 공부하고, 어떻게 자바의 싱글톤 패턴이 스프링 프레임워크에서 사용되는지 알아봅니다.
### 서비스 로직
- 서비스나 컴포넌트 등은 데이터를 담는 그릇이 아니라, 그 자체로 데이터를 처리하는 하나의 '로직'입니다.
- 그렇기에 여러개의 객체가 필요하지 않습니다.
- 오히려 요청마다 새로운 객체를 만들면 성능적으로 낭비가 될 수 있습니다.
- 따라서 서비스 로직을 담당하는 객체는 일반적으로 싱글톤 패턴을 적용합니다.

### 제어의 역전
- 제어의 역전(IoC, Inversion of Control)은 객체의 생성과 의존성 관리를 개발자가 아닌 프레임워크가 담당하는 설계 원칙입니다.
  - 스프링 컨테이너는 애플리케이션의 객체들을 생성하고 관리하며, 필요한 의존성을 주입합니다.
  - 이 과정에서 싱글톤 패턴이 자주 사용됩니다.
  - 싱글톤 패턴을 통해 컨테이너가 객체를 하나만 생성하고, 이를 애플리케이션 전반에 걸쳐 공유할 수 있습니다.
